# Repositorios del Proyecto SmartLogix

## Repositorio Principal
https://github.com/JONAHBRUZZI/smartlogix

Contiene el proyecto completo SmartLogix:
- Frontend/ — Aplicacion SPA React + Vite + TypeScript (NPM)
- Backend/ — 4 microservicios Node.js + BFF Nginx + PostgreSQL
- ENTREGABLE/ — Documentacion (analisis patrones, plan branching, repositorios)
- Docker Compose — Orquestacion multi-contenedor

---

## Componentes

### Frontend (NPM)
- **Ubicacion:** Frontend/
- **Descripcion:** SPA con React 18, TypeScript, Vite, Tailwind CSS, shadcn/ui, PWA
- **Ejecutar:** `npm install && npm run dev`
- **Build:** `npm run build`
- **Deploy:** Vercel (https://smartlogix-five.vercel.app)

### Backend For Frontend (BFF)
- **Ubicacion:** Backend/nginx/
- **Descripcion:** Nginx como API Gateway, reverse proxy y BFF
- **Puerto:** 80
- **Funcion:** Enruta peticiones del frontend a los 4 microservicios

### Microservicio: orders-service
- **Puerto:** 8081
- **Base de datos:** orders_db
- **Responsabilidad:** CRUD de pedidos, confirmacion, cancelacion, asignacion
- **Imagen Docker:** jonahbruzzi/orders-service:node

### Microservicio: inventory-service
- **Puerto:** 8082
- **Base de datos:** inventory_db
- **Responsabilidad:** Control de stock, ajustes, ventas, alertas
- **Imagen Docker:** jonahbruzzi/inventory-service:node

### Microservicio: shipping-service
- **Puerto:** 8084
- **Base de datos:** shipping_db
- **Responsabilidad:** Envios, tracking, codigo QR, confirmacion de entrega
- **Imagen Docker:** jonahbruzzi/shipping-service:node

### Microservicio: notification-service
- **Puerto:** 8085
- **Base de datos:** notification_db
- **Responsabilidad:** Persistencia y consulta de eventos de trazabilidad
- **Imagen Docker:** jonahbruzzi/notification-service:node
