Repositorios del proyecto SmartLogix — Evaluacion Parcial N°3
==============================================================

Repositorio principal (desarrollo):
  https://github.com/JONAHBRUZZI/smartlogix

Repositorio de evaluacion (Eva2):
  https://github.com/jhonabruzzi278/smartlogix-eva

Este repositorio contiene la plataforma completa de gestion logistica SmartLogix.
Incluye el frontend React (SPA + PWA), los 4 microservicios backend (Node.js/Express),
la configuracion del BFF (Nginx API Gateway) y los archivos Docker Compose para
despliegue en VM.


Descripcion de cada componente dentro del repositorio:
------------------------------------------------------

1. Frontend (React SPA)
   Ruta en el repo: /Frontend/
   Tecnologia: React 18 + TypeScript + Vite + Tailwind CSS + shadcn/ui + PWA
   Descripcion: Aplicacion web progresiva (PWA) para la gestion de pedidos,
   inventario, envios, punto de venta, clientes y reportes.
   Deploy: Vercel (https://smartlogix-five.vercel.app)

2. BFF - API Gateway (Nginx)
   Ruta en el repo: /Backend/nginx/
   Tecnologia: Nginx Alpine
   Descripcion: Reverse proxy que actua como punto unico de entrada a los
   microservicios. Enruta /api/orders, /api/inventory, /api/shipments y
   /api/notifications al servicio correspondiente.

3. orders-service (Microservicio de Pedidos)
   Ruta en el repo: /Backend/orders-service/
   Tecnologia: Node.js 22 + Express 4 + PostgreSQL
   Puerto: 8081 | Base de datos: orders_db
   Descripcion: Gestiona el ciclo de vida de los pedidos: creacion, confirmacion
   (orquesta inventory + shipping), cancelacion, asignacion de transportista y
   eliminacion.

4. inventory-service (Microservicio de Inventario)
   Ruta en el repo: /Backend/inventory-service/
   Tecnologia: Node.js 22 + Express 4 + PostgreSQL
   Puerto: 8082 | Base de datos: inventory_db
   Descripcion: Control de stock, catalogo de productos, ajustes atomicos de
   inventario y registro de ventas.

5. shipping-service (Microservicio de Envios)
   Ruta en el repo: /Backend/shipping-service/
   Tecnologia: Node.js 22 + Express 4 + PostgreSQL
   Puerto: 8084 | Base de datos: shipping_db
   Descripcion: Gestion de envios, generacion de numeros de tracking, control de
   etapas de despacho y notificacion de cambios al notification-service.

6. notification-service (Microservicio de Trazabilidad)
   Ruta en el repo: /Backend/notification-service/
   Tecnologia: Node.js 22 + Express 4 + PostgreSQL
   Puerto: 8085 | Base de datos: notification_db
   Descripcion: Persistencia de eventos para trazabilidad y auditoria. Registra
   cada cambio de estado en el flujo logistico.


Infraestructura:
----------------
- Docker Compose (desarrollo y produccion)
- PostgreSQL 15 Alpine (1 base de datos por microservicio)
- Docker Hub (imagenes pre-build para despliegue en VM)
- Vercel (hosting del frontend)
- VM DigitalOcean (hosting del backend en 104.248.60.29)
