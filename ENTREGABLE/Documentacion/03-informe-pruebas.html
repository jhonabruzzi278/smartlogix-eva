4. Técnicas de Testing Aplicadas

  • Mocking de base de datos: jest.fn() con mockResolvedValueOnce simula respuestas de PostgreSQL sin base de datos real.
  • Mocking de fetch (inter-service): global.fetch = jest.fn() intercepta llamadas HTTP entre microservicios (Saga pattern).
  • Factory mocks: evitan que Jest cargue módulos con dependencias externas (pg, express) durante la inicialización del mock.
  • Patrón AAA (Arrange-Act-Assert): todos los tests siguen el patrón de preparar mocks, ejecutar el endpoint vía supertest, y verificar status code + body.
  • Test de stored procedures: tests especializados que verifican el comportamiento del endpoint cuando el SP retorna diferentes valores (success: true/false, distintos error_msg).
  • Idempotencia de tests: beforeEach(() => jest.clearAllMocks()) limpia el estado entre cada test para evitar interferencias.
  • Require.main guard: if (require.main === module) previene el inicio del servidor cuando Jest importa el módulo, asegurando tests sin side effects.
